# 마케팅 매칭 플랫폼 — MVP 설계서 (v1.0)

> **이 문서는 Claude Code에 직접 전달하여 개발을 시작할 수 있는 수준의 설계서입니다.**
> 파이프라인 3~7단계의 핵심을 하나로 압축했습니다.

---

## 0. 프로젝트 개요

### Problem Statement

**자영업자는 마케터를 찾을 수는 있지만, 실력이 검증된 마케터인지 알 수 없어 돈을 날릴까 봐 불안하다.**
현재 임시방편: 블로그 후기 검색, 지인 추천, 일단 맡겨보고 실패 후 교체.
이로 인해 평균 1~3회의 시행착오와 수십~수백만 원의 매몰 비용이 발생한다.

### 핵심 가치 제안 (UVP)

**"검증된 마케터가 먼저 제안합니다. 포트폴리오와 실제 후기로 비교하고, 안심하고 선택하세요."**

### 사용자 Pain 우선순위

| 순위 | Pain | Level | MVP 해결 여부 |
|------|------|-------|-------------|
| 1 | 마케터 실력 검증 불가 | 5 (즉시 결제 가능) | ✅ Must |
| 2 | 적정 시세 모름 → 바가지 불안 | 4 (임시방편 사용 중) | ✅ 구조적 해결 (복수 제안 비교) |
| 3 | 진행 상황 관리 불가 | 4 | ⬜ Should (v2) |
| 4 | 마케터 찾는 채널 부재 | 3 | ✅ 플랫폼 자체가 해결 |

## 1. 전략 브리프

### 타겟 사용자

**자영업자 (의뢰자)**
- 업종: 전 업종 (음식점, 카페, 학원, 미용실, 병원, 쇼핑몰 등)
- 공통점: 마케팅 필요성은 느끼지만, 직접 할 시간/역량이 없음
- 예산: 월 30만~300만 원 수준
- 핵심 감정: 불안 + 좌절 ("또 돈 날리면 어쩌지")

**마케터 (제공자)**
- 프리랜서 마케터, 1~5인 마케팅 에이전시
- 분야: SNS, 블로그/SEO, 네이버 플레이스, 광고 대행 등 전 분야
- 핵심 니즈: 안정적 일감 확보, 포트폴리오 기반 영업

### 매칭 방식: 역경매형 (의뢰 등록 → 마케터 제안)

자영업자가 "나 이런 마케팅 필요해"를 올리면, 마케터들이 제안서를 보내는 구조.

**이 방식을 선택한 이유:**
1. 자영업자가 갑(甲) 포지션 → 바쁜 사장님이 마케터를 찾아다닐 필요 없음
2. 복수 제안 → 자연스럽게 시세 파악 (Pain #2 동시 해결)
3. 제안 시 포트폴리오 + 후기가 첨부 → 검증이 자연스럽게 작동 (Pain #1 해결)

### 경쟁사 대비 차별화

| 항목 | 숨고/크몽 | 우리 플랫폼 |
|------|----------|-----------|
| 분야 | 전 분야 (마케팅은 일부) | 마케팅 전문 (전문성 집중) |
| 검증 | 별점 + 짧은 리뷰 | 포트폴리오 (Before/After 데이터) + 상세 후기 |
| 매칭 | 범용 카테고리 | 마케팅 분야/업종별 맞춤 매칭 |
| 시세 | 불투명 | 제안 비교로 자연스럽게 형성 |

### OKR

- **Objective**: 자영업자가 검증된 마케터를 안심하고 선택할 수 있는 플랫폼 구축
- **KR1**: MVP 출시 후 4주 내 의뢰 등록 50건 달성
- **KR2**: 의뢰 1건당 평균 제안 3건 이상
- **KR3**: 매칭 성사율 30% 이상 (의뢰 대비 실제 계약)

## 2. 비즈니스 모델

### 수익 모델 (MVP 단계)

**중개 수수료 모델 (마케터 측 과금)**

| 항목 | 내용 |
|------|------|
| 모델 | 매칭 성사 시 마케터에게 수수료 부과 |
| 요율 | 계약 금액의 10% (업계 표준 5~15%) |
| 과금 시점 | 자영업자가 제안 수락 후 |
| Free Tier | 마케터 프로필 등록 무료, 월 3건 제안 무료, 이후 건당 과금 또는 구독 |
| 자영업자 | 완전 무료 (공급 측 과금으로 수요 측 진입 장벽 제거) |

**선택 근거:**
- 자영업자가 돈을 내면 의뢰 자체가 줄어듦 → 마케터도 떠남 (치킨&에그)
- 마케터는 일감을 얻는 대가로 수수료 지불 의사가 높음
- 숨고/크몽도 동일 구조 검증 완료

### Lean Canvas

| 블록 | 내용 |
|------|------|
| 문제 | 마케터 실력 검증 불가 / 시세 불투명 / 찾는 채널 부재 |
| 고객 | 전 업종 자영업자 (월 마케팅 예산 30~300만 원) |
| UVP | "검증된 마케터가 먼저 제안합니다" |
| 솔루션 | 의뢰 등록 → 포트폴리오+후기 기반 제안 비교 → 선택 |
| 채널 | 자영업자 커뮤니티(아프니까 사장이다 등), 네이버 카페, 인스타 광고 |
| 수익원 | 매칭 수수료 10% (마케터 측) |
| 비용 | 서버/인프라, 초기 마케터 모집 비용 |
| 핵심 지표 | 매칭 성사 수 (North Star) |
| 차별화 | 마케팅 전문 + 포트폴리오/성과 데이터 검증 |

### Flywheel (성장 선순환)

```
자영업자 의뢰 등록
    ↓
마케터 제안 증가 (일감이 있으니까)
    ↓
자영업자 선택지 풍부 → 만족도 UP
    ↓
후기 축적 → 플랫폼 신뢰도 UP
    ↓
더 많은 자영업자 유입
    ↓ (반복)
```

**Engine of Growth**: Sticky (리텐션 기반)
- 마케터: 후기/포트폴리오가 쌓일수록 떠나기 어려움
- 자영업자: 이전 마케터 히스토리가 있으니 재의뢰 시 플랫폼 복귀

## 3. MVP 기능 설계 (MoSCoW)

### Must (P0) — 이것 없으면 핵심 루프가 안 돌아감

| ID | 기능명 | 설명 | 해결하는 Pain |
|----|--------|------|-------------|
| M1 | 회원가입/로그인 | 자영업자/마케터 역할 선택 가입 (이메일 기반) | 진입 |
| M2 | 마케터 프로필 | 전문 분야, 경력, 포트폴리오(이미지+설명), 가격 범위 등록 | 검증 |
| M3 | 의뢰 등록 | 자영업자가 업종, 마케팅 분야, 예산, 요구사항 작성 | 찾기 + 시세 |
| M4 | 제안 보내기 | 마케터가 의뢰에 제안서 작성 (견적, 작업 계획, 포트폴리오 첨부) | 매칭 |
| M5 | 제안 비교/선택 | 자영업자가 받은 제안들을 비교하고 수락 | 검증 + 시세 |
| M6 | 후기 작성 | 작업 완료 후 자영업자가 마케터에게 후기/평점 작성 | 검증 |

### Should (P1) — 있으면 UX 향상

| 기능명 | 설명 |
|--------|------|
| 마케터 검색/탐색 | 분야별, 평점별 마케터 목록 브라우징 |
| 채팅/메시지 | 제안 수락 전 마케터와 사전 소통 |
| 알림 | 새 의뢰, 새 제안, 수락 등 알림 |
| 대시보드 | 마케터/자영업자 각각의 현황 요약 |

### Could (P2) — 나중에

| 기능명 | 설명 |
|--------|------|
| 에스크로 결제 | 플랫폼 내 안전 결제 |
| 작업 진행 관리 | 마일스톤 기반 진행 상황 트래킹 |
| AI 매칭 추천 | 의뢰 내용 기반 마케터 자동 추천 |
| 마케터 인증 배지 | 플랫폼 자체 실력 인증 시스템 |

### Won't (이번엔 안 함)

| 기능명 | 제외 이유 |
|--------|----------|
| 결제 시스템 | MVP 단계에서는 외부 결제 (계좌이체 등) |
| 분쟁 중재 | 운영 리소스 필요, MVP에서 불필요 |
| 모바일 앱 | 웹 반응형으로 충분, 앱 개발은 PMF 이후 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름: 의뢰 등록 → 제안 비교 → 매칭 성사

```
[자영업자 여정]

가입 → 역할 선택(자영업자) → 의뢰 작성
    → 업종 선택
    → 마케팅 분야 선택 (SNS/블로그/플레이스/광고/기타)
    → 예산 범위 선택
    → 상세 요구사항 작성
    → 의뢰 등록 완료 ← ⭐ 아하 모먼트: "이렇게 올리면 마케터가 알아서 연락 오는 거야?"
    → 제안 도착 알림
    → 제안 목록 확인 (포트폴리오 + 후기 + 견적 비교)
    → 제안 수락
    → (외부에서 작업 진행)
    → 후기 작성
```

```
[마케터 여정]

가입 → 역할 선택(마케터) → 프로필 작성
    → 전문 분야 선택
    → 경력/소개 작성
    → 포트폴리오 등록 (이미지 + 설명 + 성과 데이터)
    → 프로필 완성 ← ⭐ 아하 모먼트: "여기에 내 작업물 올려놓으면 일감이 들어오는 거구나"
    → 의뢰 피드 탐색 (내 분야 필터)
    → 의뢰 상세 확인
    → 제안서 작성 (견적, 계획, 포트폴리오 첨부)
    → 제안 수락 알림
    → (외부에서 작업 진행)
```

### 아하 모먼트 목표

- 자영업자: 의뢰 등록 후 **24시간 내 첫 제안 도착** (초기에는 운영팀이 마케터에게 직접 알림)
- 마케터: 프로필 등록 후 **48시간 내 첫 의뢰 확인** (초기에는 운영팀이 의뢰 시드)
- 가입 → 아하 모먼트: **5분 이내** (가입 2분 + 의뢰/프로필 작성 3분)

### 예외 경로 처리

| 상황 | 처리 |
|------|------|
| 의뢰에 제안 0건 | 48시간 후 "마케터에게 추가 노출 중" 안내 + 의뢰 수정 유도 |
| 마케터에게 의뢰 0건 | 프로필 개선 가이드 제공 + "이런 의뢰가 최근 올라왔어요" 추천 |
| 제안 수락 후 취소 | 24시간 이내 무료 취소, 이후 패널티 없음 (MVP) |
| 로딩 지연 | 스켈레톤 UI |
| 에러 | 친절한 메시지 + 재시도 버튼 |

### 테이블 관계

- users 1:1 marketer_profiles (마케터만)
- marketer_profiles 1:N portfolios
- users(client) 1:N requests
- requests 1:N proposals
- marketer_profiles 1:N proposals
- proposals 1:1 reviews

### RLS 정책

| 테이블 | 정책 | 조건 |
|--------|------|------|
| users | 본인 데이터 읽기/수정 | auth.uid() = id |
| marketer_profiles | 누구나 읽기, 본인만 수정 | SELECT: true / UPDATE: auth.uid() = user_id |
| portfolios | 누구나 읽기, 본인만 CRUD | SELECT: true / INSERT/UPDATE/DELETE: 본인 marketer_id |
| requests | 누구나 읽기(open), 본인만 수정 | SELECT: status='open' OR client_id=auth.uid() / UPDATE: client_id=auth.uid() |
| proposals | 의뢰 주인 + 제안 마케터만 | client가 request의 owner이거나, marketer 본인 |
| reviews | 누구나 읽기, 작성자만 쓰기 | SELECT: true / INSERT: auth.uid() = client_id |
| event_logs | 본인만 쓰기, 읽기 불가 | INSERT: auth.uid() = user_id |

### 핵심 화면 상세 설계

#### 랜딩페이지 (/)

**목적**: 3초 안에 "이 서비스가 뭔지" 파악 + 가입 유도

**레이아웃**:
- Hero: 헤드카피 + 서브카피 + CTA 2개 ("의뢰 등록하기" / "마케터로 가입하기")
- 신뢰 섹션: 등록된 마케터 수, 매칭 성사 수, 평균 평점
- 작동 방식: 3단계 (의뢰 등록 → 제안 비교 → 선택)
- 마케터 샘플: 상위 마케터 3~4명 카드
- CTA 반복

**헤드카피 안**:
- A안 (공포): "마케팅 대행, 또 돈만 날리실 건가요?"
- B안 (해방): "검증된 마케터가 먼저 제안합니다"

#### 의뢰 등록 (/requests/new)

**목적**: 자영업자가 3분 안에 의뢰를 올릴 수 있어야 함

**필드 구성**:
1. 업종 선택 (드롭다운: 음식점/카페/학원/미용실/병원/쇼핑몰/기타)
2. 마케팅 분야 (체크박스: SNS 마케팅/블로그·SEO/네이버 플레이스/광고 대행/종합 마케팅/기타)
3. 월 예산 범위 (슬라이더 또는 셀렉트: 30만 이하/30~50만/50~100만/100~200만/200~300만/300만 이상)
4. 의뢰 제목 (텍스트, placeholder: "예: 강남 카페 인스타그램 마케팅")
5. 상세 요구사항 (텍스트에리어, 가이드 질문 제공)

**가이드 질문 (placeholder)**:
"현재 상황, 원하는 목표, 참고할 계정이나 사례가 있다면 알려주세요"

**4가지 상태**:
- Default: 폼 표시
- Loading: 등록 중 스피너
- Empty: N/A (폼 화면)
- Error: 필드별 인라인 에러 + "다시 시도해주세요"

#### 마케터 프로필 (/marketers/:id)

**목적**: 자영업자가 "이 마케터에게 맡겨도 될까?" 판단할 수 있는 모든 정보

**레이아웃**:
- 상단: 프로필 사진 + 이름 + 전문 분야 태그 + 평점·후기 수 + 경력 + 가격 범위
- 중단: 포트폴리오 그리드 (이미지 카드, 클릭 시 상세)
- 하단: 후기 목록 (평점 + 내용 + 작성자 업종 + 날짜)

#### 제안 비교 (/requests/:id)

**목적**: 받은 제안들을 한눈에 비교

**레이아웃**:
- 상단: 내 의뢰 요약 (제목, 분야, 예산)

  - 정렬: 최신순 / 가격순 / 평점순

## 9. 기능 명세서 (Feature Specs)

### M1: 회원가입/로그인

**정상 흐름**:
| Step | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | /signup 접근 | 역할 선택 화면 표시 ("자영업자" / "마케터") |
| 2 | 역할 선택 | 가입 폼 표시 (이메일, 비밀번호, 이름) |
| 3 | 정보 입력 + 가입 버튼 | Supabase Auth 가입 처리 |
| 4 | 가입 완료 | 역할별 대시보드로 리다이렉트 |

**입력 제한**:
- 이메일: 유효한 이메일 형식
- 비밀번호: 최소 8자, 영문+숫자
- 이름: 최소 2자

**예외 처리**:
| 상황 | 에러 메시지 |
|------|-----------|
| 이메일 중복 | "이미 가입된 이메일입니다" |
| 비밀번호 형식 | "영문과 숫자를 포함하여 8자 이상 입력해주세요" |
| 네트워크 에러 | "연결이 불안정합니다. 다시 시도해주세요" |

### M2: 마케터 프로필

**정상 흐름**:
| Step | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | /profile/edit 접근 | 프로필 편집 폼 |
| 2 | 전문 분야 선택 (복수) | 태그 선택 UI |
| 3 | 경력, 소개, 가격 범위 입력 | 실시간 미리보기 |
| 4 | 저장 | DB 저장 + 성공 Toast |

**Acceptance Criteria**:
- 전문 분야 최소 1개 선택 필수
- 소개글 최소 50자
- 포트폴리오 최소 1개 등록 시 프로필 "완성" 상태

### M3: 의뢰 등록

**정상 흐름**:
| Step | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | /requests/new 접근 | 의뢰 작성 폼 |
| 2 | 업종, 분야, 예산, 제목, 설명 입력 | 실시간 유효성 검사 |
| 3 | 등록 버튼 | DB 저장 + 의뢰 상세 페이지로 이동 |
| 4 | 의뢰 등록 완료 | "마케터들에게 전달되었습니다!" 안내 |

**Acceptance Criteria**:
- 모든 필수 필드 입력 시에만 등록 버튼 활성화
- 등록 후 즉시 의뢰 피드에 노출
- 기본 마감일: 14일 후 자동 설정

### M4: 제안 보내기

**정상 흐름**:
| Step | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 의뢰 피드에서 의뢰 클릭 | 의뢰 상세 + "제안하기" 버튼 |
| 2 | "제안하기" 클릭 | 제안 작성 폼 |
| 3 | 견적, 기간, 내용, 포트폴리오 첨부 | 유효성 검사 |
| 4 | 제안 발송 | DB 저장 + 자영업자에게 알림 + requests.proposal_count 증가 |

**Acceptance Criteria**:
- 같은 의뢰에 중복 제안 불가
- 견적 금액 필수
- 제안 내용 최소 100자

### M5: 제안 비교/선택

**정상 흐름**:
| Step | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 내 의뢰 상세 페이지 | 받은 제안 카드 리스트 |
| 2 | 제안 카드 클릭 | 제안 상세 (마케터 프로필 + 포트폴리오 + 후기 포함) |
| 3 | "이 마케터 선택하기" 클릭 | 확인 모달 |
| 4 | 확인 | proposal.status='accepted' + request.status='in_progress' + 나머지 proposals 'rejected' |

**Acceptance Criteria**:
- 제안 수락 시 확인 모달 필수 ("선택하시면 다른 제안은 자동 거절됩니다")
- 수락 후 마케터 연락처 공개
- request 상태 자동 변경

### M6: 후기 작성

**정상 흐름**:
| Step | 사용자 행동 | 시스템 반응 |

| 3 | 등록 | DB 저장 + marketer_profiles.rating_avg 재계산 + review_count 증가 |

**Acceptance Criteria**:
- 별점 필수, 내용 최소 20자
- 의뢰당 후기 1개만 (중복 작성 불가)
- 후기 등록 후 수정 불가 (MVP)
